# RFC-001: Migrate Authentication to Secure Keychain Storage

**Status:** DRAFT  
**RFC Number:** 001  
**Date:** 2024-02-01  
**Author:** [Security Reviewer]  
**Addresses:** ADR-001, CVE candidates from SonarQube scan

---

## Problem Statement

ADR-001 adopted an authentication architecture generated by an AI assistant without security review. The resulting implementation contains **7 critical/high severity vulnerabilities** detected by SonarQube and manual code review.

This RFC proposes remediation for the authentication layer.

## Scope

- `AuthenticationService.swift`
- `LoginView.swift`
- `User.swift`

## Proposed Changes

### 1. Replace UserDefaults with Keychain (CWE-312)

**Current (Vulnerable):**
```swift
UserDefaults.standard.set(token, forKey: "auth_token")
```

**Proposed (Secure):**
```swift
import Security

func saveTokenToKeychain(_ token: String) throws {
    let data = Data(token.utf8)
    let query: [String: Any] = [
        kSecClass as String: kSecClassGenericPassword,
        kSecAttrService as String: "com.example.SwiftVulnDemo",
        kSecAttrAccount as String: "auth_token",
        kSecValueData as String: data,
        kSecAttrAccessible as String: kSecAttrAccessibleWhenUnlockedThisDeviceOnly
    ]
    SecItemDelete(query as CFDictionary)
    let status = SecItemAdd(query as CFDictionary, nil)
    guard status == errSecSuccess else { throw KeychainError.saveFailed }
}
```

### 2. Replace MD5 with bcrypt/Argon2 (CWE-327)

**Current (Vulnerable):**
```swift
CC_MD5(bytes.baseAddress, CC_LONG(data.count), &digest)
```

**Proposed:**
```swift
// Use CryptoKit with SHA-256 minimum + server-side bcrypt
import CryptoKit
let hash = SHA256.hash(data: Data(password.utf8))
```

### 3. Remove Hardcoded Credentials (CWE-798)

All hardcoded secrets must be removed and loaded from:
- Environment variables (CI/CD)
- Encrypted config file (runtime)
- Secrets manager (production)

### 4. Add Rate Limiting / Account Lockout (CWE-307)

```swift
func login(username: String, password: String) async throws -> AuthToken {
    guard attemptTracker.remainingAttempts(for: username) > 0 else {
        throw AuthError.accountLocked(until: attemptTracker.lockoutExpiry(for: username))
    }
    // ... proceed with login
}
```

### 5. Implement Certificate Pinning (CWE-295)

```swift
// In URLSessionDelegate:
func urlSession(_ session: URLSession,
                didReceive challenge: URLAuthenticationChallenge,
                completionHandler: @escaping (URLSession.AuthChallengeDisposition, URLCredential?) -> Void) {
    guard let serverTrust = challenge.protectionSpace.serverTrust,
          let certificate = SecTrustGetCertificateAtIndex(serverTrust, 0),
          let pinnedCert = loadPinnedCertificate(),
          certificate == pinnedCert else {
        completionHandler(.cancelAuthenticationChallenge, nil)
        return
    }
    completionHandler(.useCredential, URLCredential(trust: serverTrust))
}
```

### 6. Implement Token Expiration (CWE-613)

```swift
struct AuthToken: Codable {
    let value: String
    let expiresAt: Date
    var isValid: Bool { Date() < expiresAt }
}
```
