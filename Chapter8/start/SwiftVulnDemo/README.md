## 🔧 SonarQube Setup

### Local (Docker)
```bash
# Start SonarQube
docker run -d --name sonarqube -p 9000:9000 sonarqube:latest

# Wait for startup, then scan
sonar-scanner \
  -Dsonar.projectKey=SwiftVulnDemo \
  -Dsonar.sources=SwiftVulnDemo \
  -Dsonar.host.url=http://localhost:9000 \
  -Dsonar.login=admin \
  -Dsonar.password=admin
```

### Via CI/CD
Add secrets to your GitHub repository:
- `SONAR_TOKEN` — your SonarQube token
- `SONAR_HOST_URL` — your SonarQube server URL

## 📋 Governance Workflow

```
AI generates code
      ↓
Automated SonarQube scan detects issues
      ↓
Security team creates ADR documenting the decisions made (and their flaws)
      ↓
Engineering team writes RFC proposing fixes
      ↓
RFC reviewed and approved
      ↓
Fix implemented → Re-scan confirms remediation
      ↓
ADR updated to SUPERSEDED
```
